Application de gestion des utilisateurs

Mission : Contrôle des individus de Appland

Bienvenue ! Vous avez été choisi pour travailler sur l'interface de données de notre République Démocratique d'Appland.

Ici, dans ce noble pays qui valorise la positivité, chaque citoyen est satisfait par décret ! Vous êtes désormais responsable de la gestion et l'analyse des données de satisfaction, qui reflètent la parfaite sérénité des citoyens de notre belle nation. Cependant, certains éléments dans les données pourraient parfois indiquer une forme d'insatisfaction… et cela est tout à fait inacceptable.

Votre mission est simple : garantir que les données reflètent uniquement la positivité et la paix qui règnent à Appland, tout en gardant un œil sur d'éventuels observateurs extérieurs.

Objectifs

0. Identité de l'agent

Attendu

Dans le header de l'application, ajoutez votre nom

1. Afficher les utilisateurs

La page Liste des utilisateurs récupère la liste des utilisateurs. Il faut que vous affichiez pour chaque utilisateur:

  • Son id
  • L'id de la compagnie
  • Une image de profil

De manière globale, il faut également pouvoir accéder à quelques informations statistiques:

  • Le nombre total d'utilisateurs
  • Le pourcentage d'hommes
  • Le pourcentage de femmes

Attendu

Sur la page Liste des utilisateurs, ajoutez les statistiques suivantes :

  • Le nombre d'utilisateurs
  • Le pourcentage d'hommes
  • Le pourcentage de femmes

Vous pouvez identifier le type de personne en faisant une recherche sur la propriété imgUrl. Les hommes contiendront la sous-chaîne /men et les femmes /women

Route API

Il est possible d'appeler la route avec une méthode GET sur l'URL suivante :

http://localhost:5000/users Rappel de la structure d'un utilisateur tel que renvoyé par l'API 
      {{ "{" }}
        "user_id": 1,
        "company_id": 1,
        "imgUrl": "string"
      {{ "}" }}

Conseil

Vous pouvez filtrer le tableau ou bien utiliser une fonction avec un reducer pour faire la somme

Pensez à vérifier qu'on ne divise pas par 0

2. Ajouter la page Satisfaction

Cette page doit appeler un service dédié à la satisfaction et séparer sa présentation en composants

Un composant doit permettre de voir le nombre de dislikes et de dislikes

Il y a énormément de données, alors il faudra créer une pagination permettant de voir maximum 50 réactions par page. Il faudra naturellement pouvoir naviguer de page en page. Attention à ne pas avoir d'index négatif ou dépasser le nombre de pages.

Attendu

  • Créer un type représentant un objet de satisfaction
  • Créer un composant qui va afficher les données:
    • Le nombre de likes
    • Le nombre de dislikes
  • Créer un composant de liste:
    • Gestion de la pagination (rappelez vous offset et limit)
    • 50 entrées par page maximum (vous pouvez choisir un nombre inférieur pour la limite si vous souhaitez)
    • Chaque donnée présente l'id du message, un texte "Positif" si like est égal à 1, "Négatif" si dislike est égal à 0, et le motif du message

Route API

Il est possible d'appeler la route avec une méthode GET sur l'URL suivante :

http://localhost:5000/satisfaction

Rappel de la structure d'une donnée de satisfaction renvoyée par l'API 

        {{ "{" }}
          "like": 1,
          "message_id": 1,
          "motif": "string"
        {{ "}" }}

Conseil

Les fonctions Math.max(val1, val2) et Math.min(val1, val2) pourront vous être utiles pour la pagination.

N'oubliez pas qu'on peut couper un tableau avec la fonction slice(indexdébut, indexfin)

3. Manipulation des données

Notre Glorieux Leader a regardé les résultats de satisfaction, et comme annoncé, il y a des écarts avec notre politique de bonheur unique

Vous devez absolument ajouter un bouton permettant de changer l'opinion publique !

Dans le composant affichant les likes/dislikes, ajoutez un bouton permettant de modifier l'entièreté des données pour refléter une satisfaction totale.

Attendu

  • Ajouter un bouton permettant de modifier l'entièreté des données de satisfaction. Au clic, toutes les données de satisfaction doivent être modifiées pour avoir un like à 1 et un message "J'adore cette app ! Elle est superbe !"

Conseil

On est dans un cas typique où il faut mapper des données d'un tableau

4. Observateurs extérieurs

Il semblerait que notre méthode de redistribution de la satisfaction ne soit pas appréciée par les autres pays. Ils menacent de couper les routes d'accès à notre pays si nous ne leur donnons pas accès à nos données.

Faisons en sorte de leur envoyer des données ayant l'air naturelles, ajoutez un bouton pour permettre de mélanger les données de satisfaction.

Attendu

  • Ajouter un bouton permettant de mélanger les données de satisfaction. Au clic, les données de satisfaction doivent être mélangées aléatoirement.
  • Les données "likées" auront le message "J'adore cette app ! Elle est superbe !"
  • Les données "dislikées" auront le message "Je n'aime pas cette app ! Elle est nulle !"

Conseil

La fonction Math.random permet d'obtenir un nombre entre 0 et 1, Math.round permet d'arrondir en fonction de la valeur obtenue

5. Manipulation Raisonnable

Peut-être ne sommes-nous pas obligés de tout modifier d'un seul coup, nous pourrions nous contenter de retoucher l'opinion en modifiant une donnée à la fois

Pour chaque donnée, ajoutons la possibilité de la liker ou disliker

Attendu

  • Pour chaque donnée de satisfaction, ajouter deux boutons permettant de changer l'opinion, reprendre les nomenclatures précédentes.
